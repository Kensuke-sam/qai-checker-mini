# システム移行計画書

## 概要

本文書はオンプレミスからAWSへの移行計画を記述する。
対象サーバ台数: 12台
移行期間: 2024年4月〜2024年9月（6ヶ月）
総予算: 15,000,000円

## 前提条件

- 現行システムの稼働率は99.9%を維持する
- データ移行量は約2.5TB
- ダウンタイムは最大4時間とする

## 移行フェーズ

### Phase 1: 準備（4月〜5月）

1. 現行環境のインベントリ作成
2. ネットワーク設計
3. セキュリティポリシー策定
4. AWSアカウントのセットアップ

### Phase 2: 検証（6月〜7月）

1. ステージング環境の構築
2. データベース移行テスト
3. アプリケーション動作確認
4. パフォーマンステスト実施

### Phase 3: 本番移行（8月〜9月）

1. 本番データのバックアップ
2. DNSの切り替え
3. 動作確認・監視強化
4. 旧環境の停止

## 技術スタック

- コンピュート: EC2 (m5.xlarge)
- ストレージ: S3, EBS
- データベース: RDS (PostgreSQL 15)
- 監視: CloudWatch
- IaC: Terraform

## リスク管理

| リスク | 影響度 | 対策 |
|--------|--------|------|
| データ欠損 | 高 | 二重バックアップ |
| 性能劣化 | 中 | 事前ベンチマーク |
| コスト超過 | 中 | 月次コストレビュー |
